# ResView v1.1.15 – Changelog der gesamten v1.1.x-Serie

**Veröffentlicht:** Mai 2026  
**VS Code Mindestversion:** 1.85.0

---

## Zusammenfassung aller Neuerungen seit v1.1.0

### v1.1.0 – Panel überlebt Editor-Reload

Das ResView-Panel blieb nach einem VS Code Reload (`Developer: Reload Window`) zuvor grau und leer. Ein `WebviewPanelSerializer` wird jetzt beim Aktivieren der Extension registriert — VS Code stellt das Panel danach automatisch wieder her, inklusive gespeichertem UI-Zustand (Gerät, Kategorie, Zoom, Ausrichtung).

---

### v1.1.4 – Grid-Overlay, Pixel-Ruler, Inspector Mode, Open-in-Browser

**Grid-Overlay**  
Ein togglebares Raster (Button „Grid") wird als semi-transparente lila Linien über dem Preview-Bereich eingeblendet. Skaliert automatisch mit dem Zoom-Level.

**Pixel-Ruler**  
Horizontales und vertikales Lineal (Button „Ruler"), positioniert **außerhalb** des sichtbaren Content-Bereichs im Bezel-Rand. Tick-Marks alle 10 px, Labels alle 100 px. Wird bei Gerätewechsel, Rotation und Zoom automatisch neu berechnet.

**Inspector Mode**  
Der „Inspect"-Button aktiviert einen Element-Inspector für lokale Dev-Server. Die Extension startet beim Panel-Öffnen einen lokalen HTTP-Proxy, der ein Inspector-Script in jede HTML-Response injiziert. Beim Hover über Elementen erscheinen:
- Violetter Highlight-Rahmen
- CSS-Selektor des Elements
- Box-Model-Diagramm (margin / border / padding / content) im DevTools-Stil
- Tabelle mit Computed Styles (color, background, font-size, display, position, …) mit Farb-Swatches

Der Proxy entfernt `X-Frame-Options`- und `Content-Security-Policy`-Header, sodass auch Seiten, die normalerweise nicht in iframes ladbar sind, im Preview funktionieren.

> Einschränkung: Inspector funktioniert nur für HTTP-Localhost-Server.

**Open-in-Browser-Button**  
Der „↗"-Button öffnet die aktuell geladene URL direkt im Standardbrowser.

---

### v1.1.5 – Inspector-Panel: fixes Höhe, horizontales Properties-Layout

Das Inspector-Panel hatte eine dynamische Höhe, die bei jedem Hover-Wechsel sprang und flackerte. Jetzt: feste Höhe von **170 px** (`overflow: hidden`). Das Properties-Layout wurde von einer vertikalen 2-Spalten-Liste auf ein **horizontales 6-Spalten-Grid** umgestellt (3 Property-Value-Paare nebeneinander), was den Platz effizient nutzt und die Zeilenzahl halbiert.

---

### v1.1.6 – Zoom bis 150 %

Der Zoom-Slider wurde von maximal 100 % auf **150 %** erweitert (25–150 %, Schrittweite 5 %). Nützlich um kleinere Geräte (z. B. iPhone SE) bei höherer Vergrößerung zu betrachten.

---

### v1.1.7 – Grid: einstellbare Zellengröße

Neben dem Grid-Button gibt es jetzt ein Dropdown zur Auswahl der Rastergröße: **4 · 8 · 16 · 24 · 32 · 64 px** (Standard: 8 px). Änderungen werden live übernommen. Das Dropdown ist im inaktiven Zustand leicht ausgeblendet (Opacity 45 %).

---

### v1.1.8 – Permanenter Live-Server-Button, Port 5500 in Auto-Detection

Ein fester **„⚡ Live Server :5500"**-Button ist jetzt immer in der Toolbar sichtbar — lädt mit einem Klick `http://localhost:5500`, unabhängig vom Auto-Scan-Ergebnis. Port 5500 wurde außerdem in die automatische Server-Erkennung aufgenommen.

---

### v1.1.9–1.1.14 – Inspector: CSS-Quelldatei-Anzeige

Im Inspector-Panel wird jetzt rechts neben dem CSS-Selektor als grüner Badge angezeigt, **aus welcher CSS-Datei** die Regel stammt. Die Entwicklung dieser Funktion umfasste mehrere Iterationen:

| Version | Änderung |
|---|---|
| v1.1.9 | Grundimplementierung: `getSources()` via `document.styleSheets` + `element.matches()` |
| v1.1.10 | Bugfix: HTML-Escaping für Dateinamen wie `<style>` (wurden vom Browser als Tag interpretiert); Badge-Größe erhöht |
| v1.1.11 | Inline-`<style>`-Blöcke ohne `href` werden zunächst übersprungen |
| v1.1.12 | Fallback: Inline-Blöcke zeigen den HTML-Dateinamen (`index.html`) der aktuellen Seite |
| v1.1.13 | Vite/React-Support: `data-vite-dev-id`-Attribut wird ausgelesen — CSS-Imports wie `App.css` werden korrekt erkannt statt als `index.html` |
| v1.1.14 | Bugfix: `@media`-Blöcke wurden ignoriert (kein `selectorText`). `scanRules()` durchsucht jetzt rekursiv alle verschachtelten Regelmengen (`@media`, `@supports`, `@layer` …) |

**Aktuelle Priorität der Quellermittlung:**

| Situation | Angezeigter Name |
|---|---|
| Externes Stylesheet (`<link href="…">`) | CSS-Dateiname, z. B. `main.css` |
| Vite-injizierter `<style data-vite-dev-id="…">` | CSS-Dateiname, z. B. `App.css` |
| Sonstiger Inline-`<style>`-Block | HTML-Dateiname, z. B. `index.html` |

---

### v1.1.15 – Hot Reload im Inspector Mode

CSS-Änderungen werden im Inspector Mode jetzt **sofort** übernommen ohne den Inspector aus- und wieder einschalten zu müssen. Vite's HMR läuft über WebSocket — der Proxy hat bisher nur HTTP weitergeleitet, wodurch die HMR-Verbindung unterbrochen wurde. Der Proxy fängt jetzt `upgrade`-Events ab und tunnelt WebSocket-Verbindungen per TCP direkt zum Vite-Dev-Server durch.
