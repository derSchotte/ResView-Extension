# ResView v1.1.4 – Release Notes

**Veröffentlicht:** Mai 2026  
**VS Code Mindestversion:** 1.85.0

---

## Neuerungen in dieser Version

### Grid-Overlay

Ein togglebares 8px-Raster kann über den **„Grid"-Button** in der Toolbar aktiviert werden. Das Raster wird als semi-transparente lila Linien über dem Preview-Bereich eingeblendet und skaliert automatisch mit dem Zoom-Level. Nützlich zum Prüfen von Ausrichtung und Abständen im Layout.

### Pixel-Ruler

Über den **„Ruler"-Button** werden horizontale und vertikale Lineale eingeblendet. Die Lineale positionieren sich **außerhalb des sichtbaren Frame-Bereichs** — d.h. sie überlagern nicht den Inhalt, sondern erscheinen im Bezel-Rand des Geräts. Tick-Marks alle 10 px, Labels alle 100 px in echten CSS-Pixeln (0, 100, 200, …). Bei Gerätewechsel, Rotation und Zoom-Änderung werden sie automatisch neu berechnet und neu positioniert.

### Inspector Mode (localhost only)

Der **„Inspect"-Button** aktiviert einen Element-Inspector für lokale Dev-Server. Technisch startet die Extension beim Öffnen des Panels einen leichtgewichtigen HTTP-Proxy auf einem freien Port. Beim Aktivieren des Inspectors lädt der iframe die Seite über den Proxy, der ein Inspector-Script in jede HTML-Response injiziert.

**Beim Hover über Elementen erscheint:**
- Ein violetter Highlight-Rahmen um das Element
- Der CSS-Selektor des Elements (`button.nav-link#home`)
- Ein **Box-Model-Diagramm** (margin / border / padding / content) im Firefox DevTools-Stil
- Eine Tabelle mit den wichtigsten Computed Styles: color, background, font-size, font-family, display, position, border-radius u.a. — mit Farb-Swatches für Farbwerte

Der Proxy entfernt außerdem `X-Frame-Options`- und `Content-Security-Policy`-Header, sodass auch Seiten, die normalerweise nicht in einem iframe ladbar sind, im Preview funktionieren.

> **Einschränkung:** Der Inspector funktioniert nur für HTTP-Localhost-Server. Externe HTTPS-Sites werden nicht unterstützt (würde ein MITM-Zertifikat erfordern).

### „Open in Browser"-Button

Ein **„↗"-Button** neben dem Rescan-Button öffnet die aktuell geladene URL direkt im Standard-Systembrowser.

## Technische Details

- **Proxy:** Node.js `http.createServer` auf `127.0.0.1:0` (zufälliger freier Port). Leitet alle Requests an den konfigurierten Target-Host weiter; injiziert Inspector-Script via `</body>`-Replace in HTML-Responses.
- **Inspector-Script:** Vanille-JS, inline injiziert. Nutzt `mouseover`-Events + `window.getComputedStyle()` + `window.parent.postMessage()` für die Kommunikation mit dem Webview.
- **Ruler:** `position: absolute` in `#deviceShell` (außerhalb `#deviceBezel`). Position wird per JS anhand der Bezel-Padding-Werte berechnet, sodass die Lineale bündig an der Content-Kante beginnen.
- **Grid:** CSS `repeating-linear-gradient` als `background-image` auf einem `pointer-events: none`-Overlay.
