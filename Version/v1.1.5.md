# ResView v1.1.5 – Release Notes

**Veröffentlicht:** Mai 2026  
**VS Code Mindestversion:** 1.85.0

---

## Neuerungen in dieser Version

### Inspector Panel – kein Höhen-Flackern mehr

**Problem in v1.1.4:** Beim Hovern über verschiedene Elemente im Inspector Mode änderte sich die Anzahl der angezeigten CSS-Properties je nach Element. Da das Panel eine dynamische Höhe hatte (`max-height: 200px`), sprang die Höhe beim Wechsel zwischen Elementen ständig hin und her — was zu starkem Flackern der gesamten Oberfläche führte.

**Fix:** Das Inspector-Panel hat jetzt eine **feste Höhe von 170px** (`height: 170px; overflow: hidden`). Egal wie viele Properties ein Element besitzt, die Panel-Höhe bleibt konstant.

### Inspector Panel – Properties horizontal angeordnet

Zusätzlich wurde das Layout der CSS-Properties von einer einfachen vertikalen 2-Spalten-Liste (`Eigenschaft | Wert`) auf ein **horizontales 6-Spalten-Grid** (`Eigenschaft Wert | Eigenschaft Wert | Eigenschaft Wert`) umgestellt. Dadurch werden bis zu 3 Property-Value-Paare nebeneinander angezeigt, was den verfügbaren horizontalen Platz rechts neben dem Box-Model-Diagramm effizient nutzt und die benötigte Zeilenzahl halbiert.

```
Vorher:  color       #ffffff          (1 Paar pro Zeile, viele Zeilen → variable Höhe)
         background  #1e1e2e
         font-size   14px
         ...

Nachher: color #fff  |  background #1e1e2e  |  font-size 14px   (3 Paare pro Zeile, fixe Höhe)
         display flex  |  position relative  |  border-radius 6px
```
