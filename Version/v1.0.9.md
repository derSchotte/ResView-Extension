# ResView v1.0.9 – Release Notes

**Veröffentlicht:** Mai 2026  
**VS Code Mindestversion:** 1.85.0

---

## Neuerungen in dieser Version

### Eigene Geräte verwalten
Benutzer können jetzt eigene Geräte (Smartphones, Tablets, Desktops) anlegen, benennen und dauerhaft speichern. Die gespeicherten Geräte erscheinen in einer eigenen Gruppe „★ Custom" in der Geräteauswahl und bleiben auch nach dem Schließen von VS Code erhalten. Einzelne eigene Geräte können über den Löschen-Button (🗑) jederzeit wieder entfernt werden.

### UI-Zustand wird gespeichert
Die Einstellungen der Extension werden beim Schließen automatisch gespeichert und beim nächsten Öffnen wiederhergestellt:
- Ausgewählte Kategorie (Phone / Tablet / Desktop)
- Ausgewähltes Gerät
- Zoom-Stufe
- Ausrichtung (Portrait / Landscape)

Die zuletzt geladene URL wird bewusst nicht gespeichert.

### Korrekte Bezel-Rotation (Portrait ↔ Landscape)
Der Geräterahmen und der Notch drehen sich bei Portrait/Landscape-Wechsel nun korrekt mit. Im Landscape-Modus wechselt der breite Rahmen (Notch-Seite) von oben nach links, der Notch wandert entsprechend.

### Kein sichtbarer Scrollbalken im Vorschauframe
Auf echten Mobilgeräten ist kein nativer Scrollbalken sichtbar. Die Extension bildet dieses Verhalten jetzt korrekt nach: Der iframe wird intern um die Breite des nativen Scrollbalkens vergrößert und durch den Wrapper-Container abgeschnitten, sodass der Scrollbalken außerhalb des sichtbaren Bereichs liegt.

### Kein Leerraum unter dem Gerät
Das Zoomen per CSS `transform: scale()` hinterlässt normalerweise den vollen unskalierten Layout-Raum. Dieser überschüssige Platz unterhalb des Geräts wird jetzt durch eine negative `marginBottom`-Kompensation vollständig entfernt.

### Korrigierte Gerätedaten
Die physischen Gerätedaten (CSS-Viewport und PPI) wurden für alle 24 eingebauten Geräte auf Richtigkeit geprüft und korrigiert:
- Samsung Galaxy S23: PPI 393 → 425
- OnePlus 12: Höhe 919 → 905 CSS-px
- Samsung Galaxy Tab S9 Ultra: PPI 188 → 240, Viewport 960×1848 → 1232×1973
- Google Pixel Tablet: PPI 210 → 276, Viewport 960×1280 → 800×1280

## Bekannte Einschränkungen

- Externe URLs (z. B. `https://example.com`) können in der Vorschau aufgrund von `X-Frame-Options`- oder `Content-Security-Policy`-Headern der Zielseite blockiert sein.
- Der Inhalt des Vorschau-iFrames ist aus Sicherheitsgründen (Cross-Origin) nicht direkt vom Webview aus zugänglich.

---

## Technische Details

- **Kompilierungsschritt:** TypeScript → JavaScript (`out/`)
- **Persistenz:** VS Code `globalState` (global, workspaceunabhängig)
- **Scrollbalken-Unterdrückung:** iframe-Verbreiterung + `overflow: hidden` auf dem Wrapper
- **Zoom:** CSS `transform: scale()` mit negativer `marginBottom`-Kompensation
