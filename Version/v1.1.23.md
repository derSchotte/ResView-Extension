# v1.1.23 – 2026-05-26

## Touch-Emulation

### Added
- **Touch emulation.** Mouse events inside the preview iframe are automatically mapped to synthetic `TouchEvent`s (`touchstart`, `touchmove`, `touchend`, `touchcancel`). Swipe gestures, touch-only menus, mobile sliders and any library that listens to raw touch events now respond correctly without a real touch device. The emulation runs unconditionally — no toggle required.
- **`navigator.maxTouchPoints` spoofing.** The injected script sets `navigator.maxTouchPoints = 1` so that libraries which gate touch-code on this property (Hammer.js, Swiper, etc.) treat the page as a touch-capable device.
- **Touch script always injected via proxy.** The proxy now injects a lightweight touch-emulation script into every HTML response, regardless of whether inspector mode is active. Inspector script injection remains conditional on inspector mode.

### Fixed
- **Inspector / CSS Rules broken after enabling touch routing.** Once all pages were routed through the proxy, toggling the inspector sent an `inspectorReady` message with the same proxy URL that was already loaded. Chromium/Electron treats `iframe.src = sameUrl` as a no-op, so the inspector script was never injected. Fixed by removing the `src` attribute before each `inspectorReady` navigation to guarantee a fresh page load every time.
