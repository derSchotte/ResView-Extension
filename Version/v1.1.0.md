# ResView v1.1.0 – Release Notes

**Veröffentlicht:** Mai 2026  
**VS Code Mindestversion:** 1.85.0

---

## Neuerungen in dieser Version

### Panel überlebt Editor-Reload

**Problem in v1.0.x:** Nach einem Fenster-Reload (`Developer: Reload Window`) oder einem vollständigen Editor-Neustart zeigte das ResView-Panel nur noch ein leeres graues Fenster. Der Nutzer musste das Panel schließen und manuell über `Strg+Shift+R` neu öffnen.

**Ursache:** VS Code versucht beim Start alle zuvor offenen Webview-Panels wiederherzustellen. Ohne einen registrierten `WebviewPanelSerializer` weiß VS Code nicht, wie es den Inhalt neu aufbauen soll — das Panel bleibt leer.

**Fix:** In `extension.ts` wird beim Aktivieren der Extension ein `WebviewPanelSerializer` registriert. VS Code ruft daraufhin nach einem Reload automatisch `deserializeWebviewPanel` auf, was das Panel vollständig neu initialisiert — inklusive gespeichertem UI-Zustand (Gerät, Kategorie, Zoom, Ausrichtung).

```
Vorher:  Reload → graues Fenster → manuell Strg+Shift+R
Nachher: Reload → Panel erscheint automatisch wieder
```

---

## Geänderte Dateien

| Datei | Änderung |
|---|---|
| `src/extension.ts` | `registerWebviewPanelSerializer` beim Aktivieren registrieren |
| `src/ResViewPanel.ts` | `viewType` von `private` auf `public` geändert; neue statische Methode `revive()` |

---

## Technische Details

- `vscode.window.registerWebviewPanelSerializer(viewType, serializer)` — teilt VS Code mit, wie ein Panel dieses Typs nach einem Reload wiederhergestellt werden soll
- `revive(panel, context)` — nimmt das von VS Code bereits angelegte (aber leere) Panel entgegen und führt dieselbe Initialisierung durch wie bei `createOrShow()`
- Der gespeicherte UI-Zustand (via `globalState`) wird dabei wie gewohnt wiederhergestellt
