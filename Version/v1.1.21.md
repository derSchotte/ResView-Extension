# v1.1.21 – 2026-05-26

## Comprehensive code refactoring – Clean Code, OOP & ES Modules

### Changed
- **TypeScript backend fully restructured.** The extension backend has been split into dedicated modules: `src/types.ts` (shared type definitions), `src/constants.ts` (all magic strings and config keys), `src/storage.ts` (Repository pattern for devices and UI state), `src/WebviewHtml.ts` (Builder pattern for webview HTML/CSP generation). All private fields and methods use consistent naming (no underscore prefix) with TypeScript private modifiers.
- **Frontend split into native ES modules.** `media/main.js` (one monolithic file) has been replaced by 12 focused ES module files loaded natively by the browser — no bundler required: `app.js`, `appState.js`, `bridges.js`, `constants.js`, `utils.js`, `eventEmitter.js`, and six controller modules under `media/controllers/`.
- **Observer / EventEmitter pattern introduced.** A lightweight `EventEmitter` base class (`media/eventEmitter.js`) drives all internal communication. `AppState` extends it and emits typed events (`deviceChanged`, `deviceApplied`, `persistState`, `closeCssRules`). Controllers subscribe instead of calling each other directly.
- **Repository pattern for storage.** `DeviceRepository` and `UiStateRepository` encapsulate all `globalState` reads and writes, removing scattered storage calls from `ResViewPanel`.
- **Command pattern for message dispatch.** `ResViewPanel.handleMessage()` routes each inbound webview message type to a dedicated handler method (`onReady`, `onRescan`, `onOpenExternal`, `onAddDevice`, `onDeleteDevice`, `onSaveUiState`, `onInspectorToggle`), replacing a single large switch block.
- **CSP tightened.** The Content Security Policy now uses a cryptographically secure nonce (`crypto.randomBytes(16)`) for the two entry scripts, and adds `webview.cspSource` to `script-src` so ES module imports from the media directory are allowed without `'unsafe-inline'` or `'strict-dynamic'`.
- **URL parsing modernised.** All uses of the deprecated `url.parse()` in the proxy have been replaced with `new URL()` (Node.js built-in WHATWG URL API).
- **XSS hardening in CSS rules panel.** CSS property names and values — which originate from the inspected page and are untrusted — are now inserted using `textContent`/`createTextNode` rather than `innerHTML`. Color swatches use `element.style.background = value` (a style assignment, not HTML injection).
- **Error message sanitisation in proxy.** Upstream error responses no longer include `err.message` in the body, preventing internal stack traces or host details from leaking to the webview.
- **`onOpenExternal` URL scheme validation.** The handler that opens URLs in the system browser now rejects any scheme other than `http:` or `https:`, preventing `file:`, `javascript:`, or custom-protocol abuse.
